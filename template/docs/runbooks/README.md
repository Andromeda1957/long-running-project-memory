# Runbooks

Runbooks are the operational checklists agents use to complete recurring work.
They preserve required global steps without bloating the agent startup context.

Use `AGENTS.md` to choose a runbook, then follow the relevant checklist.

Memory-system runbooks (shipped with this scaffold):

- `task-memory-triage.md`
- `docs-change.md`
- `memory-maintenance.md`
- `memory-review.md`

Add your own project runbooks beside these — for example `build.md`,
`deploy.md`, `release.md`, or `verification.md` — and list them in `AGENTS.md`.
